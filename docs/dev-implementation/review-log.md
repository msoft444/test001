## 重大論点（優先度付き）

### 1. [優先度: 高] うるう年・月末境界の計算正確性
- 2024-02-29, 2024-02-28→2024-03-01 等の境界値で正しく計算できること。
- JavaScriptのDateオブジェクトは自動繰り上げするため、入力バリデーションで存在しない日付（例: 2023-02-29）を弾く必要がある。

### 2. [優先度: 中] 負数・ゼロ・大数値の入力制御
- 日数入力欄に0, 負数, 非常に大きな数（例: 999999）を入力した場合の挙動を明確にする。
- 設計方針: 負数は許容し「何日前」として扱う（符号反転）。ゼロは許容（同日を返す）。上限は設けないがDate範囲外はエラー表示。

### 3. [優先度: 中] タイムゾーン一貫性
- JSのnew Date()はローカルTZ依存。日付のみ扱うため、時刻部分を00:00:00固定として差分計算時のDST影響を排除する。
- 入力はtype="date"のvalue（YYYY-MM-DD文字列）をパースし、UTC midnight基準で計算する方針を推奨。

### 4. [優先度: 低] アクセシビリティ
- ダークモード切替はaria-label付与を推奨。必須ではないが品質向上に寄与。
