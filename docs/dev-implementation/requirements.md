# 要件定義書

## 1. 目的/スコープ
- **目的**: 日付間の日数計算および特定日数後の日付算出を容易に行えるWEBアプリケーションを提供する。
- **スコープ**: ブラウザ上で完結するシングルページアプリケーション（SPA）とし、Dockerコンテナとして配布・実行可能にする。

## 2. 機能要件
- **日数差分計算機能**:
  - ユーザーが「開始日」と「終了日」を指定できる。
  - 2つの日付の差分（日数）を計算し表示する。
- **日付加減算機能**:
  - ユーザーが「基準日」と「日数（整数）」を入力できる。
  - 「何日後（加算）」か「何日前（減算）」かを選択できる。
  - 計算結果となる日付を表示する。
- **ダークモード切替機能**:
  - 画面上のスイッチで、アプリケーションの配色を「ライトモード」と「ダークモード」に切り替えられる。

## 3. 非機能要件
- **ポータビリティ**: Dockerコンテナとして構築され、環境依存なく起動できること。
- **レスポンシブ**: PCおよびスマートフォンでの表示に対応すること。
- **パフォーマンス**: 計算処理はクライアントサイドで行い、即座に結果を表示すること。

## 4. 制約・前提
- **技術スタック**: フロントエンドは標準的なHTML/CSS/JavaScriptを使用。サーバーサイドは既存環境（Python 3.11）を活用した軽量サーバー（http.server等）を想定。
- **データ永続化**: データの保存は行わない（ステートレス）。

## 5. 主要テストシナリオ
- 正常系: 2つの日付（例: 2024-01-01と2024-01-10）を入力し、正しい差分（9日）が表示されるか。
- 正常系: 基準日（例: 2024-01-01）と日数（10日後）を入力し、正しい日付（2024-01-11）が表示されるか。
- 正常系: うるう年（例: 2024-02-29）を含む計算が正しいか。
- 正常系: ダークモード切替スイッチを押下し、背景色・文字色が反転するか。
